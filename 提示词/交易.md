# 一、整体流程说明

这四个模块的核心逻辑都是：

1. **后端**：按用户 ID + 状态 / 类型，从订单 / 商品表中统计数量、查询列表
2. **前端**：请求接口获取数据，渲染图标、文字、数量，点击跳转对应列表页

# 二、数据库表设计（核心依赖）

先确认你的核心表结构，这是所有功能的基础：

## 1. 商品表 `goods`

sql









```
CREATE TABLE `goods` (
  `id` bigint PRIMARY KEY AUTO_INCREMENT,
  `user_id` bigint NOT NULL COMMENT '发布者ID',
  `title` varchar(100) NOT NULL,
  `price` decimal(10,2) NOT NULL,
  `status` tinyint DEFAULT 0 COMMENT '0-在售 1-已卖出 2-下架',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 2. 订单表 `orders`

sql









```
CREATE TABLE `orders` (
  `id` bigint PRIMARY KEY AUTO_INCREMENT,
  `goods_id` bigint NOT NULL,
  `seller_id` bigint NOT NULL COMMENT '卖家ID',
  `buyer_id` bigint NOT NULL COMMENT '买家ID',
  `status` tinyint DEFAULT 0 COMMENT '0-待付款 1-已付款/待发货 2-已发货 3-已完成/待评价 4-已评价',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 1. 统计接口（用于首页展示 4 个数字）

java



运行







```
@RestController
@RequestMapping("/api/trade")
public class TradeController {

    @Autowired
    private TradeService tradeService;

    @GetMapping("/count")
    public Result<TradeCountDTO> getTradeCount(@RequestParam Long userId) {
        TradeCountDTO count = tradeService.getTradeCount(userId);
        return Result.success(count);
    }
}
```

## 2. Service 核心实现（最关键）

java



运行







```
@Service
public class TradeService {

    @Autowired
    private GoodsMapper goodsMapper;
    @Autowired
    private OrderMapper orderMapper;

    public TradeCountDTO getTradeCount(Long userId) {
        TradeCountDTO dto = new TradeCountDTO();
        
        // 1. 我发布的：当前用户发布的所有商品数量（含在售/已卖出）
        Integer publishCount = goodsMapper.countByUserId(userId);
        dto.setPublishCount(publishCount);
        
        // 2. 我卖出的：当前用户作为卖家，已成交的订单数量
        Integer soldCount = orderMapper.countSoldBySellerId(userId);
        dto.setSoldCount(soldCount);
        
        // 3. 我买到的：当前用户作为买家，已成交的订单数量
        Integer boughtCount = orderMapper.countBoughtByBuyerId(userId);
        dto.setBoughtCount(boughtCount);
        
        // 4. 待评价：当前用户作为买家，状态为「已完成/待评价」的订单数量
        Integer waitCommentCount = orderMapper.countWaitCommentByBuyerId(userId);
        dto.setWaitCommentCount(waitCommentCount);
        
        return dto;
    }
}
```

## 3. JS 逻辑（请求接口 + 跳转）

运行







```
Page({
  data: {
    userId: 1, // 当前登录用户ID
    tradeCount: {
      publishCount: 0,
      soldCount: 0,
      boughtCount: 0,
      waitCommentCount: 0
    }
  },

  onShow() {
    this.getTradeCount()
  },

  // 1. 获取交易统计数据
  getTradeCount() {
    wx.request({
      url: 'https://your-domain.com/api/trade/count',
      data: { userId: this.data.userId },
      success: (res) => {
        this.setData({
          tradeCount: res.data.data
        })
      }
    })
  },

  // 2. 跳转对应列表页

    wx.navigateTo({ url: '/pages/trade/my-publish/my-publish' })
```

# 五、关键功能说明与扩展

表格







| 功能     | 核心逻辑                                                     | 扩展优化                                       |
| -------- | ------------------------------------------------------------ | ---------------------------------------------- |
| 我发布的 | 查询 `goods` 表中 `user_id = 当前用户` 的所有商品            | 支持按状态（在售 / 已卖出 / 下架）筛选         |
| 我卖出的 | 查询 `orders` 表中 `seller_id = 当前用户` 的已成交订单       | 支持按订单状态（待发货 / 已发货 / 已完成）筛选 |
| 我买到的 | 查询 `orders` 表中 `buyer_id = 当前用户` 的已成交订单        | 同上，可增加筛选功能                           |
| 待评价   | 查询 `orders` 表中 `buyer_id = 当前用户` 且 `status = 待评价` 的订单 | 评价后更新订单状态，数量自动减少               |

# 六、常见问题排查

1. **数字不更新**：检查 `onShow` 生命周期是否调用了 `getTradeCount`，确保每次进入页面都刷新数据
2. **SQL 统计为 0**：确认 `user_id` 传参正确，订单 / 商品表中的 `seller_id`/`buyer_id` 字段是否正确存储
3. **跳转白屏**：检查 `app.json` 中是否注册了跳转的页面路径
